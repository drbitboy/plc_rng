Gaussian-distribution Random Number Generator (GNG/RNG)

N.B. This is designed to generate a distribution with a mean of zero and a variance of unity.

N.B. The natural logarithm approximation algorithm rungs only work with U1 values in the range [1/32768:sqrt(2)]

N.B. The sine approximation algorithm rungs only work with argument in the range [-0.25:+1.25]

N.B. The natural logarithm and sine approximations are accurate to about six significant digits; see ../../verify/ for more details

N.B. A 16-bit INTEGER (word) LCG with (a,b) = (20077,12345) is mathematically correct, but arithmetic errors in precluded its use here

- GNG uses the Box-Muller Transform

- Cf. https://en.wikipedia.org/wiki/Box%E2%80%93Muller_transform

- Test harness is [RSLogix Micro Starter Lite + RSLogix Emulator 500],

- Which has neither LN nor COS, nor SIN instructions

- A pair of uniform-distribution random numbers (UDRNs) generate the inputs: U1 and U2

- Uniform RNG uses a Linear Congruential Generator (LCG)

- Cf. https://en.wikipedia.org/wiki/Linear_congruential_generator

- LCG Seed algorithm sequence is [UDRN(i+1) = (UDRN(i) * a + b) MOD m]

- The following info regarding overflow is likely no longer true since the code switched to 32-bit LONGs in the LCGs

- [Overflow Trap] bit (S:5/0) must be cleared after LCG calculations to prevent Halt on Minor Fault

- 16-bit MUL and ADD, with [Math Overflow Selected] bit (S:2/14) set, implements effective modulus [m] of 32,768 (2**15)

- Sign (bit 15; 0x8000) of 16-bit seed is irrelevant to calculation

- Bitwise OR with 16-bit -32768 (0x8000) ensures all output LCG output seeds are negative i.e. [-32768:0)

- Scaling seeds via division by -32768.0 yields U1 and U2 values in range (0.0:1.0]

- This is critical for U1, which cannot be 0.0 as that would cause a fault in the LN instruction

- The UDRN pair is used to generate a pair of Gaussian-Distribution Random Numbers (GDRNs)

- GNG1 = sqrt(-2 log(U1)) * cos(2 PI U2)

- GNG2 = sqrt(-2 log(U1)) * sin(2 PI U2)

- Each scan produces one Gaussian-distribution Random Number

- Both UDRNs are required for the first GDRN (first scan)

- The UDRNs do not change for the second GDRN (second scan) of a pair

- There will be alternating pairs of scans, each pair comprises an even scan first, followed by an odd scan

- B3:0.0 is ODDEVEN: 0=>even scan; 1=>odd scan

- Both UDRNs of the UDRN pair will be generated during an even (first) scan in a pair, along with the first GDRN

- There will be some other setup done during an even scan e.g.

- Scaling both UDRN seeds to U1 and U2 in the range (0.0:1.0]

- Calculating intermediate values that are the same for both even and odd scans

- The odd (second) scan in a pair will generate only the second GDRN, using the UDRN pair

N.B. All [B3:9/*] outputs are non-functional bits used to connect per-rung comments to the correct rung
